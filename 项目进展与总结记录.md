### 2026-03-10 认证授权模块开发 (Day 7-8)
**当前所处阶段**：Day 7-8: 图形验证码、登录/登出、Sa-Token RBAC 系统性集成
**完成的主要工作**：
1. **认证模块 DTO/VO 层编码**：
   - `LoginDTO`：登录请求参数（含 Jakarta Validation 校验注解）
   - `LoginVO`：登录成功返回体（Token、用户基本信息）
   - `CaptchaVO`：验证码响应体（Base64图片+唯一Key）
2. **认证核心 Service (`AuthServiceImpl`)**：
   - Hutool `LineCaptcha` 图形验证码生成，存入 Redis 5分钟有效
   - BCrypt 密码散列比对
   - 防爆破机制：连续失败 3 次自动锁定账号 1 小时，锁定过期自动解除
   - Sa-Token `StpUtil.login()` 签发 Token
   - 异步 `@Async` 记录登录日志至 `sys_login_log`
3. **认证 Controller (`AuthController`)**：
   - `GET /api/auth/captcha` — 获取图形验证码
   - `POST /api/auth/login` — 用户登录
   - `POST /api/auth/logout` — 用户登出
4. **RBAC 数据源 (`StpInterfaceImpl`)**：
   - 实现 Sa-Token 的 `StpInterface` 接口
   - 从数据库 `sys_user_role → sys_role → sys_role_menu → sys_menu` 链式查询用户角色标识与权限标识
5. **全局异常处理升级**：
   - `GlobalExceptionHandler` 新增 `NotLoginException`(401)、`NotPermissionException`(403)、`NotRoleException`(403) 精确拦截
   - `campus-common` POM 新增 `sa-token-core` 依赖
6. **主启动类增强**：
   - `CampusSystemApplication` 添加 `@EnableAsync`（支持异步日志）和 `@EnableScheduling`（支持后续定时任务）
