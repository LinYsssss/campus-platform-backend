# 一体化智慧校园系统 - 56天每日工作计划安排表（纯后端版）

## 1. 编制说明
本表依据《后端详细开发方案》与《工作计划安排》进行细化颗粒度拆解，剥离了前端页面交互工作，专为**后端研发人员**制定。旨在将8周（56天）的项目里程碑下发到每一个具体的工作日，确保后端接口开发、表结构设计、中间件搭建及压测任务进度透明、责任到天。

## 2. 第1周：项目启动与基建搭设阶段 (Day 1 - Day 7)
*   **Day 1**：召开项目需求宣贯会，熟悉《需求分析文档》及后端技术规格说明；确立后端代码仓库（Git/Gitee）及分库开发规范。
*   **Day 2**：后端全局骨架初始化（Java 17 + Spring Boot 3.x + Maven多模块设计）。
*   **Day 3**：数据库物理模型设计，完成 `campus_system` 库创建及底层管控组（`sys_`）的基础表 DDL 编写。
*   **Day 4**：继续数据库建设，完成教研链组（`edu_`）和后勤服务组（`campus_`）表结构设计及主外键约束解耦层面的逻辑设计。
*   **Day 5**：后端基础环境脚手架集成（MyBatis-Plus 通用配置、Sa-Token 权限中间件引入及 Hutool 工具包应用）。
*   **Day 6**：后端 MyBatis-Plus 代码生成器配置与运行，生成所有大模块的 `Controller`、`Service`、`Mapper` 及继承 `BaseEntity` 的基础实体类。
*   **Day 7**：统一 RESTful API 报文响应实体（`Result<T>`）规范制定、基于 `@RestControllerAdvice` 的 `GlobalExceptionHandler` 全局异常拦截方案开发定型。

## 3. 第2周：核心管控底座与鉴权中心开发 (Day 8 - Day 14)
*   **Day 8**：开发图形验证码生成接口与核心安全登录/登出接口，并集成 Hutool 的 BCrypt 加密验证机制。
*   **Day 9**：实现 Sa-Token 全局路由拦截器配置（`SaTokenConfig`），打通多级角色/权限验证的底层控制逻辑。
*   **Day 10**：实现**用户台账管理模块**的后端 CRUD 接口（增删查改及基于逻辑删除的启停用、拉黑控制）。
*   **Day 11**：实现**用户数据 Excel 批量导入导出**接口后端校验逻辑，利用 `@Validated` 及实体注解拦截并清洗非法数据。
*   **Day 12**：开发系统**参数化字典台账**及**菜单权限树形查配**的后端模块树状数据组装接口。
*   **Day 13**：运用 AOP 切面技术开发**操作日志注解**（`@LogRecord`），并利用 `@Async` 异步无阻挠记录敏感业务变更。
*   **Day 14**：后端核心底座自自测收尾；对外输出 API 结构文档，配合对接平台登录与动态路由提取接口的数据验核。

## 4. 第3周：教研核心模块（一）：课程与资料共享 (Day 15 - Day 21)
*   **Day 15**：开启教研业务域；实现**课程管理体系**（开课表创建、权限分配及行政班级映射）的后端持久层与 Service 开发。
*   **Day 16**：完善课表数据条件刷取、复杂分页查询接口；联调多对多对应关系的接口传输DTO和VO展示对象转换。
*   **Day 17**：开发基于本地磁盘路径配置的**电子教案 IO 上传与查阅模块**（实现白名单格式拦截、容量限制以及MD5验重）。
*   **Day 18**：深入 Mapper 层书写多表长连缀 MyBatis XML，供给师生双向多跨度的**课表视图动态渲染数据拉取接口**。
*   **Day 19**：实现基于内部流转状态机的**结课量化文字评价**模块开发及落盘逻辑。
*   **Day 20**：联调“授课架构-课表查询-资料下载”后端微闭环；补写单元格测试，修复空指针拦截及越权查阅漏洞。
*   **Day 21**：后端代码第一阶段内部 Review（Milestone 1），开展慢 SQL 初步规避及项目 Service 间相互依赖梳理。

## 5. 第4周：教研核心模块（二）：动态考勤与请假网络 (Day 22 - Day 28)
*   **Day 22**：攻坚核心高频缓存；设计基于 Redis/JetCache 的**限时签到考勤池 (Cache-Aside)** 结构设计及缓存生命周期淘汰规则。
*   **Day 23**：开发教师端“定时考勤任务”下发建池接口，以及教师端监控在线考核状况的实时统计获取口。
*   **Day 24**：高强度攻坚：开发**学生端防单点穿击、瞬时秒签口**，利用缓存阻遏连击请求并通过内存与异步线程缓冲持久化。
*   **Day 25**：启动请假审批流架构：开发**电子化请假工单提出**接口、校验逻辑及对应的附件举证图片保存接口。
*   **Day 26**：开发背靠请假数据的**审批树接单处理及状态扭转接口**（通过Service逻辑阻拦越级或越权批复）。
*   **Day 27**：交叉查询考勤签到库与请假有效流水池，构建基于时间窗的当日班级**考勤率宏观快照生成口**。
*   **Day 28**：采用简易并发脚本进行自身接口多线程加压探测（校验锁卡与双重签到破防情况）；完成本周后管架构小结。

## 6. 第5周：教研核心模块（三）：防改成绩与系统通告 (Day 29 - Day 35)
*   **Day 29**：利用 Apache POI / EasyExcel 研发**多维度成绩空白模板发放**及基于线下成绩单的防抖动、校验引擎导入接口。
*   **Day 30**：开发**成绩防篡改递审流转机制**后端中枢（在 Service 层织入状态封死拦截，阻断 `status=2` 时的物理层面的 Update 动作）。
*   **Day 31**：搭建**最高管理员成绩二审落锤与打回重修数据冲销节点**验证口。
*   **Day 32**：开发面向全服或单模块的历次成绩、排名大面板的高效查询接口（建立合理的联合索引支持检索）。
*   **Day 33**：建立带影像举证链接的带审改分数复议/申诉工单生成流程，以及消息分发机制。
*   **Day 34**：切换回公共版块：研发多维度全景业务标签投放的**富文本通告与站内信批处理下发接口**。
*   **Day 35**：打通用户前端信息提示红点轮询口或 WebSocket 信道口，交付全部布告的阅读状态扭转（未读->已读）更新接口。

## 7. 第6周：后勤园区服务与超级宏观看板 (Day 36 - Day 42)
*   **Day 36**：启动服务类组建：搭建宿舍区域架构表字典集；开发带有影像链结的**工单报送接口与流水号规则发生器**。
*   **Day 37**：开发宿管派单系统——工单接收反馈流、承接更新等强状态机推进型交互接口及最终验收点评打分口。
*   **Day 38**：外挂模拟仿真对接，实现**校园卡消费实时清算流水账接口读取与模拟额度冻结口**实现（MOCK）。
*   **Day 39**：图书类仿真接入，搭建**检索接口与限期催归超期账号限流小模块核心管控口**。
*   **Day 40**：开启大屏供数挑战：废除前端实时查库，设立由 `@Scheduled(cron="...")` 支撑的定时聚合引擎扫描庞大明细表。
*   **Day 41**：通过聚合引擎算出当日全院签到曲线、后勤修复率等重度结构树，**作为 JSON 快照固化入 JetCache 缓存堆**。
*   **Day 42**：暴露大屏专用极致极速视图查询口，对接全聚合无运算的快照分发，优化大数据吞吐量及响应时效（<= 150ms）。

## 8. 第7周：全链路大联调、集成压测与除错修复 (Day 43 - Day 49)
*   **Day 43**：API 代码进入收敛管控期（Feature Freeze），开展**全流程端到端的后端接口串联集成自测**。
*   **Day 44**：针对前置依赖和状态传递（如请假联动考勤，考勤流转结课等）进行黑盒输入测试验伤。
*   **Day 45**：重度测试执行：利用 **JMeter 配置高压高频并发打卡大兵团压跑测试（200-500并发）**，拉取服务器各项指标监测卡死状况。
*   **Day 46**：依据压测探针数据，集中修复 Service 处理阻塞池、缓存击穿、乃至数据库隔离级别引发的幽灵数据。
*   **Day 47**：实行延宕 SQL 精减优化行动：查杀全表扫描；在高频读少频写接口周围设立 Caffeine + Redis 双防守阵型。
*   **Day 48**：全局回归全套 API，确保底层基座与实体修补过程没撕出别的连环坍塌。
*   **Day 49**：清理工程开发级硬编码测试位点废弃代码；整理出高度可用的全量干净数据库 DDL 及必选字典 DML 初始化载入脚本。

## 9. 第8周：封版打包、文档沉淀与项目交付上线 (Day 50 - Day 56)
*   **Day 50**：后端工程封箱锁库。Maven Profile 切换出 Prod 配置资源包，用 Lifecycle 构建准生产版本独立可执行的 Fat-Jar 产物。
*   **Day 51**：预发服务器运维配置开荒：搭建 Linux JDK 17 底座支撑、持久稳固 MySQL 集簇搭建预配及 Redis 容器加载。
*   **Day 52**：进行预演级别的 Fat-Jar 试部署，验证应用层与 Nginx 的反向代理及日志切片归集能否通畅跑达。
*   **Day 53**：编写交付期必备技术文献集：《后端部署及灾备手册汇》以及最终校准版的全接口列表。
*   **Day 54**：为上线前期前端的边缘错误抛单或现场预演中发生的漏点提供线上实时微量 Hotfix 。
*   **Day 55**：技术盘点结案：配合项目全景演示确保后端守护进程无假死无宕机；封存运维启动 Shell 手册。
*   **Day 56**：向甲方开发/运维组交接完整的后端 Repository 树底版、建库集及权限口令。项目全面收刀结项落幕。
